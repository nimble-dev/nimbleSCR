Wolverine Example

Cyril Milleret, Daniel Turek and Pierre Dupont

2020-08-03

This vignette demonstrates the wolverine spatial capture recapture model, as in “Efficient MCMC for Spatial Capture-Recapture Models” (Turek et al, submitted). Specifically, we implement the final version of the wolverine model and MCMC using nimble (de Valpine et al. 2017; NIMBLE Development Team 2020). Details of the functions and procedure are provided therein.

Load Libraries

library(nimble)
library(basicMCMCplots)
library(coda)

Load nimbleSCR package

We compiled all functions in the nimbleSCR package version 0.1.0 (Bischof et al. 2020).

library(nimbleSCR)

Define Model Structure

Here, we define the nimble model structure.

code <- nimbleCode({
  ## priors
  psi ~ dunif(0, 1)
  sigma ~ dunif(0, 50)
  p0 ~ dunif(0, 1)
  ## loop over individuals
  for(i in 1:n.individuals) {
    ## AC coordinates
    sxy[i,1] ~ dunif(0, x.max)
    sxy[i,2] ~ dunif(0, y.max)
    ## habitat constraint 
    ones[i] ~ dHabitatMask( s = sxy[i,1:2],
                            xmin = lowerCoords[1],
                            xmax = upperCoords[1],
                            ymin = lowerCoords[2],
                            ymax = upperCoords[2],
                            habitat = habitat.mx[1:y.max,1:x.max])
    ## latent dead/alive indicators
    z[i] ~ dbern(psi)
    ## likelihood
    y[i, 1:nMaxDetectors] ~ dbinom_sparseLocalSCR( detNums = nbDetections[i],
                                                   detIndices = yDets[i,1:nMaxDetectors],
                                                   size = trials[1:n.detectors],
                                                   p0 = p0,
                                                   s = sxy[i,1:2],
                                                   sigma = sigma,
                                                   trapCoords = detector.xy[1:n.detectors,1:2],
                                                   localTrapsIndices = detectorIndex[1:n.cells,1:maxNBDets],
                                                   localTrapsNum = nDetectors[1:n.cells],
                                                   resizeFactor = ResizeFactor,
                                                   habitatGrid = habitatIDDet[1:y.maxDet,1:x.maxDet],
                                                   indicator = z[i])
  }
  ## derived quantity: total population size
  N <- sum(z[1:n.individuals])
})

Load and Process Data

We load the wolverine example data available from the Dryad Digital Repository (Milleret et al. 2018). See Milleret et al. (2019) for a complete description of the data.

The data file can be downloaded at [https://doi.org/10.5061/dryad.42m96c8].

We also create objects code, constants, data, and inits for later use in the function nimbleModel.

load("WolverineData.RData")
data <- list(y = my.jags.input$y,
             z = my.jags.input$z,
             detector.xy = my.jags.input$detector.xy,
             habitat.mx = my.jags.input$habitat.mx,
             ones = my.jags.input$OK,
             lowerCoords = c(0,0),
             upperCoords = c(
               dim(my.jags.input$habitat.mx)[2],
               dim(my.jags.input$habitat.mx)[1]),
             trials = rep(1, dim(my.jags.input$detector.xy)[1]))
constants <- list(n.individuals = my.jags.input$n.individuals,
                  n.detectors = dim(my.jags.input$detector.xy)[1],
                  y.max = dim(my.jags.input$habitat.mx)[1],
                  x.max = dim(my.jags.input$habitat.mx)[2])
inits <- list(sxy = inits.1$sxy,
              z = inits.1$z,
              p0 = 0.05,
              psi = 0.5,
              sigma = 6)

Create custom distribution to increase MCMC efficiency

The dbinom_sparseLocalSCR() distribution incorporates three features to increase computation efficiency.

1. Local detector evaluation

This step consists in restricting calculations linked to detection function (here a halfnormal function) to detectors within a maximum realistic radius (Milleret et al. 2019). We use the function getLocalTraps to identify the set of detectors that are within a certain distance \(d_{max}\) of each habitat cell center (blue points in the plot below). These reduced sets of detectors are stored in the localTrapIndices matrix and are later used in the local evaluation of the detection model to speed-up calculations. The value of \(d_{max}\) should be as small as possible but always large enough so that for any particular individual, the sxy values of the initial activity center location restrict the calculation of \(p\) to all detectors with positive detections. The \(d_{max}\) value will therefore affect the number of columns in localTrapIndices Here, we use \(d_{max}=38\).

We also resize the habitat matrix to lower dimensions before identifying the local detectors. This reduces the number of habitat cells for which we have to identify the set of detectors that are within \(d_{max}\) of the cell center. The goal is to create the object localTrapsIndices of the smallest dimension possible for a more efficient processing of the model.
Here, we resize the habitat matrix by a factor of 24, which corresponds to the the resizeFactor argument. This value will affect how many rows will compose localTrapIndices.

DetectorIndex <- getLocalTraps(habitatMask = data$habitat.mx,
                               trapCoords = data$detector.xy,
                               dmax = 38,
                               resizeFactor = 24)

constants$y.maxDet <- dim(DetectorIndex$habitatGrid)[1]
constants$x.maxDet <- dim(DetectorIndex$habitatGrid)[2]
constants$ResizeFactor <- DetectorIndex$resizeFactor
constants$n.cells <- dim(DetectorIndex$localTrapsIndices)[1]
constants$maxNBDets <- DetectorIndex$numLocalTrapsMax
data$detectorIndex <- DetectorIndex$localTrapsIndices
data$nDetectors <- DetectorIndex$numLocalTraps
data$habitatIDDet <- DetectorIndex$habitatGrid

2. Sparse observation matrix

We re-express the detection matrix y as a sparse matrix, to reduce its size. In this representation, we turn the detection matrix y into three objects:

  • detIndices: where each row (corresponding to one individual) contains the detector identification numbers where that individual was detected.
  • y: a second matrix of identical dimension, containing the number of detections occurring at each detector. This second matrix is necessary for modelling non-binary detections (e.g. \(binomial\) observation models)
  • detNums: a vector containing the number of detectors where each individual was detected.
ySparse <- getSparseY(x = my.jags.input$y)
data$y <- ySparse$y[,,1]  
data$yDets <- ySparse$detIndices[,,1]
data$nbDetections <- ySparse$detNums[,1]
constants$nMaxDetectors <- ySparse$maxDetNums

3. Skip unnecessary calculations

The function dbinom_sparseLocalSCR() takes the logical argument indicator that specifies whether the individual i is available (\(z_i\) = 1) for detection or not (\(z_i\) = 0). When \(z_i\) = 0, calculations of \(p_i\) are not performed and therefore increases MCMC efficiency.

Configure and Build MCMC

We configure an MCMC algorithm to the Rmodel model object.

We assign MCMC monitors to \(N\), \(\sigma\), and \(p_0\).

Block sampling to increase MCMC efficiency

We also remove the univariate Metropolis-Hastings samplers which were assigned (by default) to each AC component, the sxy variables. Instead, we add joint Metropolis-Hastings samplers (RW_block) samplers to each \(x\) and \(y\) coordinate pair sxy[i, 1:2].

conf <- configureMCMC(Rmodel, monitors = c("N", "sigma", "p0"), print = FALSE)
conf$removeSamplers("sxy")
ACnodes <- paste0("sxy[", 1:constants$n.individuals, ", 1:2]")
for(node in ACnodes) {
  conf$addSampler(target = node,
                  type = "RW_block",
                  control = list(adaptScaleOnly = TRUE),
                  silent = TRUE)
}
Rmcmc <- buildMCMC(conf)

Compile and Run MCMC

Finally, we compile both the model and MCMC objects and execute the compiled MCMC for 10 000 iterations.

Cmodel <- compileNimble(Rmodel)
## compiling... this may take a minute. Use 'showCompilerOutput = TRUE' to see C++ compilation details.
## compilation finished.
Cmcmc <- compileNimble(Rmcmc, project = Rmodel)
## compiling... this may take a minute. Use 'showCompilerOutput = TRUE' to see C++ compilation details.
## compilation finished.
MCMC_runtime <- system.time(
  samples <- runMCMC(Cmcmc, niter = 50)
)
## running chain 1...

Results

First, we can extract the MCMC runtime (0.1 minutes in this case):

round(MCMC_runtime[3] / 60, 1)

##     0.1

Next, we can check the posterior effective sample size (ESS) resulting from our 10 000 posterior samples for the three parameters we tracked (\(N\), \(\sigma\), and \(p_0\)):
